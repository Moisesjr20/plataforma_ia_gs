I want to build a conversational AI SaaS platform that helps users interagir com três assistentes de IA com personalidades e funções diferentes. It should feel limpo, refinado e elegante and be optimized for responsive web (desktop and mobile).

Project Name:
GE IA

Target Audience:
• Profissionais que desejam interagir com IAs especializadas
• Equipes comerciais buscando automação por IA
• Usuários explorando assistentes de IA personalizados

Core Features and Pages:
✅ Página de Login e Cadastro
• Tela inicial com login via email e senha
• Link para cadastro de novos usuários com campos básicos (nome, email, senha)
• Validação simples de entrada
• Após login, redirecionamento automático para a página de recepção

✅ Página de Recepção (Lobby de IAs)
• Exibe três cards com os agentes de IA disponíveis:

Augusto: Agente analítico e estratégico (ex: planejamento, insights)

Sofia Brander: Especialista em branding e marketing

Carol SDR: Focada em prospecção e vendas, com linguagem direta e persuasiva
• Cada card contém: nome, descrição curta da IA e botão "Conversar"

✅ Página de Chat com IA
• Ao clicar em um agente, abre uma tela de chat personalizada com o nome e a função da IA no topo
• Interface de chat estilo mensagem (bolhas), com histórico local por sessão
• Toda comunicação com a IA é enviada/recebida via webhooks do n8n (estrutura de webhook configurável para cada agente)
• Estrutura pronta para incluir integração futura com autenticação do n8n se necessário

Tech Stack (Recommended Defaults):
• Frontend: Vite, TypeScript, React, shadcn/ui components, and Tailwind CSS
• Backend & Storage: Supabase
• Auth: Email/password login (Supabase Auth)

Design Preferences:
• Font: Inter
• Colors:

Primary: #1C1C1E (grafite escuro refinado)

Accent: #5C7CFA (azul elegante para realces e botões)

Backgrounds: branco puro e cinzas claros sutis
• Layout: Mobile-first, UI moderna baseada em cards, com bastante espaçamento e tipografia legível

Optional AI Feature:
• Cada agente de IA tem sua própria "personalidade" e tom de voz, definidos por parâmetros no webhook do n8n (ex: contexto, estilo de resposta).
• Estrutura flexível permite escalar para mais agentes no futuro com mínima adaptação.
